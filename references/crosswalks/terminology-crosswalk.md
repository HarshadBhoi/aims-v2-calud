This document maps equivalent concepts across all audit standards. Critical for the Standards Abstraction Layer.

## Abstraction Layer Implications

Based on this crosswalk, the Standards Abstraction Layer MUST support:

1. **Configurable terminology** — Every label must be driven by the standard pack, not hardcoded
2. **Variable finding elements** — 3 elements (SOX) to 5 elements (IIA), with different labels
3. **Multiple classification schemes** — MW/SD/Deficiency (GAGAS/SOX) vs Major/Minor NC (ISO) vs Critical/Major/Minor (IIA)
4. **Different report structures** — Some require explicit compliance/independence statements, others don't
5. **Different QA approaches** — Peer review vs QAIP vs PCAOB inspection vs surveillance audit
6. **Different CPE rules** — 80hrs/2yr vs 40hrs/yr vs 120hrs/3yr
7. **Different engagement types** — Some standards have advisory/consulting, others don't
8. **Positive findings** — ISO 19011 documents conformities; GAGAS only documents issues
