# 6.2 — PDF Report Engine (Multi-Standard)

**Parent**: [Phase 6 — Reporting & Analytics](OVERVIEW.md)
**Estimated effort**: 3-4 days

---

## Tasks

- [ ] **Report Engine Architecture**: Standard pack provides template → engine fills data → outputs PDF
- [ ] **Template System**: Each standard pack defines its report templates (pdfmake definitions)
- [ ] **Common Report Components**: Header, footer, cover page, TOC, page numbers, watermark (DRAFT)
- [ ] **Data Injection**: Auto-populate from database (findings, recommendations, team, scope, etc.)
- [ ] **Server-Side Generation**: Generate PDFs in NestJS (BullMQ background job for large reports)
- [ ] **Client-Side Preview**: In-browser PDF preview before download
- [ ] **Report Versioning**: Track Draft v1, v2, v3... Final
- [ ] **Report Storage**: Archive generated PDFs in S3
- [ ] **Batch Generation**: Generate multiple reports at once (e.g., all engagement reports for fiscal year)
- [ ] **Custom Branding**: Tenant logo, colors, and formatting in reports
- [ ] **DOCX Export**: Secondary export format using docx library (optional)
