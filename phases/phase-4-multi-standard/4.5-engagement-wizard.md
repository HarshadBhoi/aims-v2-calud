# 4.5 — Multi-Standard Engagement Wizard

**Parent**: [Phase 4 — Multi-Standard Framework](OVERVIEW.md)
**Estimated effort**: 2 days

---

## Objective

Enhance the engagement creation wizard to support selecting multiple standards for a single engagement.

---

## Tasks

### 4.5.1 — Multi-Standard Selection
- [ ] Step 1 of wizard: select one or more standards (multi-select chips)
- [ ] Primary standard designation (determines default terminology and report format)
- [ ] Compatible standards check (warn if standards have conflicting requirements)
- [ ] Show crosswalk coverage: "Selecting GAGAS + IIA covers 85% of both standards' requirements"

### 4.5.2 — Merged Configuration
- [ ] When multiple standards selected:
  - Finding form shows union of all required elements
  - Checklists merge (combined QA items, deduplicated via crosswalk)
  - Workflow uses the stricter standard's approval chain
  - Report can be generated per standard or as combined

### 4.5.3 — Standard-Specific Views
- [ ] Tab or toggle to view engagement through lens of each standard
- [ ] Compliance status per standard: "GAGAS: 80% complete, IIA: 72% complete"
- [ ] Generate report for specific standard from multi-standard engagement

---

## Notes

- Multi-standard engagement is the key differentiator vs competitors
- Keep UX simple — most users will pick one standard; multi is for power users
- The merged configuration logic is complex — thorough testing required
