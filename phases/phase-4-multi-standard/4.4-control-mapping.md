# 4.4 — Control Mapping (Test Once, Comply Many)

**Parent**: [Phase 4 — Multi-Standard Framework](OVERVIEW.md)
**Estimated effort**: 3 days

---

## Objective

Enable organizations to document a control once and map it to requirements across multiple standards, eliminating redundant testing.

---

## Tasks

### 4.4.1 — Unified Control Library
- [ ] Central control catalog (tenant-scoped)
- [ ] Control fields: ID, title, description, owner, type, frequency, evidence
- [ ] Tag controls to multiple standard requirements
- [ ] Control effectiveness rating

### 4.4.2 — Multi-Standard Tagging
- [ ] When documenting a control, tag it to all applicable standards:
  - "Segregation of duties in AP" satisfies:
    - GAGAS §6.33 (sufficient evidence of controls)
    - IIA Standard 2130 (control adequacy)
    - SOX AS 2201 (ICFR control design)
- [ ] Visual indicator showing which standards each control covers

### 4.4.3 — Test Once, Report Many
- [ ] Test a control once → result applies to all tagged standards
- [ ] Generate standard-specific reports from the same test data
- [ ] Coverage dashboard: show gaps where a standard has unmet requirements

### 4.4.4 — Coverage Gap Analysis
- [ ] For each standard: show requirements with no mapped controls
- [ ] Risk-prioritized gap list
- [ ] Suggested controls for uncovered requirements

---

## Notes

- This is a top-tier feature that enterprise GRC platforms charge premium for
- Directly reduces audit effort for multi-standard organizations
- The control library becomes a reusable asset across years and engagements
