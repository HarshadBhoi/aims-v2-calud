# 2.6 — Workflow & Approval Engine

**Parent**: [Phase 2 — Core Audit Engine](OVERVIEW.md)
**Estimated effort**: 3-4 days

---

## Objective

Build a configurable approval workflow engine that supports multi-step review chains. Workflow definitions come from standard packs, making the engine standard-agnostic.

---

## Tasks

### 2.6.1 — Workflow Engine Service
- [ ] `WorkflowService` in NestJS:
  - `submitForApproval(entityType, entityId, workflowCode)` — Start workflow
  - `getWorkflowStatus(entityType, entityId)` — Current status and steps
  - `approve(approvalId, comments)` — Approve current step
  - `reject(approvalId, comments, returnToStep?)` — Reject with comments
  - `delegate(approvalId, toUserId)` — Delegate to another person
  - `recall(entityType, entityId)` — Recall submission (if not yet acted on)
  - `getMyPendingApprovals(userId)` — Approval queue for a user
- [ ] Workflow reads step definitions from `StandardPack.approvalWorkflows`
- [ ] Auto-advance to next step on approval
- [ ] On rejection: return to author with comments

### 2.6.2 — Default Workflow Definitions

```typescript
// Engagement Approval (common across standards)
{
  code: "ENGAGEMENT_APPROVAL",
  steps: [
    { order: 1, label: "Supervisor Review", requiredRole: "SUPERVISOR" },
    { order: 2, label: "Director Approval", requiredRole: "DIRECTOR" },
  ]
}

// Finding Approval
{
  code: "FINDING_APPROVAL",
  steps: [
    { order: 1, label: "Senior Auditor Review", requiredRole: "SENIOR_AUDITOR" },
    { order: 2, label: "Supervisor Approval", requiredRole: "SUPERVISOR" },
  ]
}

// Report Approval (5-stage for GAGAS)
{
  code: "REPORT_APPROVAL",
  steps: [
    { order: 1, label: "Lead Auditor Review", requiredRole: "SENIOR_AUDITOR" },
    { order: 2, label: "Supervisor Review", requiredRole: "SUPERVISOR" },
    { order: 3, label: "QA Review", requiredRole: "QA_REVIEWER" },
    { order: 4, label: "Director Approval", requiredRole: "DIRECTOR" },
    { order: 5, label: "Final Sign-Off", requiredRole: "DIRECTOR" },
  ]
}
```

### 2.6.3 — Approval Center Page
- [ ] Tabs:
  - **My Pending** — Approvals assigned to current user
  - **My Submissions** — Items I submitted for approval
  - **All Pending** — All pending approvals (Supervisor+ view)
  - **History** — Completed approvals
- [ ] For each approval: entity type, title, submitted by, submitted date, current step, urgency
- [ ] Quick actions: Approve, Reject, Delegate, View
- [ ] Approve/reject with comments (dialog)
- [ ] Filters: entity type, urgency, date range

### 2.6.4 — Approval Timeline Component
- [ ] Reusable component showing workflow progress:
  ```
  ○ Submitted by John (2026-04-10)
  ● Supervisor Review — Approved by Jane (2026-04-11) ✓
  ◉ Director Approval — Pending (assigned to Director Smith)
  ○ QA Review — Waiting
  ○ Final Sign-Off — Waiting
  ```
- [ ] Color-coded: green (approved), red (rejected), blue (current), gray (waiting)
- [ ] Click step to see comments

### 2.6.5 — Notifications
- [ ] In-app notification when:
  - You receive an approval request
  - Your submission is approved/rejected
  - An approval is delegated to you
  - A pending approval is overdue (configurable threshold)
- [ ] Email notification (Phase 5.5 — placeholder hook for now)

### 2.6.6 — Workflow Customization
- [ ] Tenant admin can customize workflows:
  - Add/remove steps
  - Change required roles
  - Set optional steps
  - Add parallel approval steps (future)
- [ ] Custom workflows override standard pack defaults
- [ ] Validate workflow (must have at least one step, roles must exist)

## Notes

- The workflow engine is polymorphic — works for engagements, findings, reports, planning docs
- Approval comments are stored and shown in history (compliance requirement)
- Delegation chain must be auditable
- Consider SLA tracking: time in each approval step
