# 2.7 — Workpaper Management

> **⚠ Reconciliation 2026-04-28**: DEFERRED. No workpapers, no chain-of-custody flows, no immutability triggers in Slice A. The `Workpaper` model + bitemporal columns + `WorkpaperLink` polymorphism exist in the canonical schema but are not exercised. Resumes when a future slice introduces evidence handling.

**Parent**: [Phase 2 — Core Audit Engine](OVERVIEW.md)
**Estimated effort**: 3 days

---

## Objective

Build the workpaper (document) management system — upload, organize, review, sign-off, and version control audit evidence files.

---

## Tasks

### 2.7.1 — Workpaper CRUD API
- [ ] tRPC router: `workpaper.*`
  - `workpaper.list` — List with filters (engagement, status, reviewer, type)
  - `workpaper.upload` — Upload file + metadata
  - `workpaper.download` — Generate pre-signed download URL
  - `workpaper.update` — Update metadata
  - `workpaper.delete` — Soft delete (Lead+ only)
  - `workpaper.review` — Mark as reviewed with notes
  - `workpaper.replaceVersion` — Upload new version, archive old

### 2.7.2 — Upload Interface
- [ ] Drag-and-drop zone with multiple file support
- [ ] Upload progress bar per file
- [ ] Metadata form on upload:
  - Title (auto-filled from filename)
  - Reference number (WP-001, auto-increment)
  - Description
  - Category (Planning, Fieldwork, Evidence, Correspondence, Other)
  - Linked work program / procedure
- [ ] File type validation (configurable whitelist)
- [ ] Size limit enforcement
- [ ] Duplicate detection (same filename warning)

### 2.7.3 — Workpaper List View
- [ ] Table: Reference, Title, Type, Size, Uploaded By, Date, Status, Reviewer
- [ ] Filters: category, status (Draft, Reviewed, Approved), uploaded by, date
- [ ] Quick preview (inline for PDFs and images)
- [ ] Bulk download (zip)
- [ ] Bulk review

### 2.7.4 — Review & Sign-Off
- [ ] Review workflow:
  1. Preparer uploads workpaper (status: Draft)
  2. Reviewer opens, examines, marks as Reviewed (with notes)
  3. Supervisor approves or requests changes
- [ ] Review notes stored per workpaper
- [ ] Review status: Draft → Under Review → Reviewed → Approved → Needs Revision
- [ ] Electronic sign-off (reviewer name, date, role — logged)

### 2.7.5 — Version Control
- [ ] Upload new version while keeping previous versions
- [ ] Version history list (v1, v2, v3...) with download for each
- [ ] Compare metadata between versions
- [ ] Current version clearly marked
- [ ] Cannot delete reviewed versions (audit trail)

### 2.7.6 — Folder Organization
- [ ] Automatic folder structure per engagement:
  ```
  ENG-2026-001/
  ├── Planning/
  ├── Fieldwork/
  ├── Evidence/
  ├── Correspondence/
  └── Reports/
  ```
- [ ] Custom folders (user-created)
- [ ] Move workpapers between folders
- [ ] Folder-level access control (optional)

---

## Notes

- Workpapers are audit evidence — immutability and chain of custody matter
- Reviewed workpapers should be locked (require explicit "unlock" to modify)
- File content is in S3; metadata in PostgreSQL
- Consider full-text search of document content (PDF text extraction) for Phase 5+
- Cross-reference workpapers from findings, tests, and reports
