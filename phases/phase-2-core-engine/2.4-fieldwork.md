# 2.4 — Fieldwork & Evidence

> **⚠ Reconciliation 2026-04-28**: DEFERRED. No work programs, observations, audit tests, or sampling worksheets in Slice A. Resumes when a future slice introduces fieldwork.

**Parent**: [Phase 2 — Core Audit Engine](OVERVIEW.md)
**Estimated effort**: 3-4 days

---

## Objective

Build the fieldwork module — where auditors execute procedures, collect evidence, document observations, and escalate issues to findings.

---

## Tasks

### 2.4.1 — Observation/Issue Tracking
- [ ] CRUD API for observations (issues found during fieldwork)
- [ ] Fields: title, description, severity, status, linked procedure, evidence
- [ ] Status: Open → Under Review → Escalated to Finding | Closed (Not Significant)
- [ ] "Escalate to Finding" action — creates a Finding pre-populated from observation
- [ ] Link observations to work program procedures

### 2.4.2 — Audit Testing
- [ ] Test CRUD (linked to PRCM entries)
- [ ] Test fields:
  - Test objective
  - Control being tested
  - Test approach (inquiry, observation, inspection, re-performance)
  - Sample details (link to sampling worksheet)
  - Test results (pass/fail/exception with details)
  - Conclusion
  - Linked workpapers (evidence)
- [ ] Test status: Not Started → In Progress → Complete → Reviewed
- [ ] Aggregate pass/fail rate per engagement

### 2.4.3 — Evidence Collection
- [ ] Link any file upload to an observation, test, or finding as evidence
- [ ] Evidence types: document, screenshot, email, interview notes, data extract
- [ ] Evidence metadata: date collected, source, collected by
- [ ] Chain of custody tracking (who uploaded, who reviewed, when)
- [ ] Evidence sufficiency flag (per standard requirements)

### 2.4.4 — Fieldwork Dashboard
- [ ] Overview of fieldwork progress per engagement:
  - Work programs: % complete
  - Tests: pass/fail summary
  - Observations: open/closed/escalated
  - Workpapers: uploaded/reviewed
- [ ] Gantt-style timeline view of procedures
- [ ] Assignee workload view

### 2.4.5 — Interview & Meeting Notes
- [ ] Create interview/meeting records
- [ ] Fields: date, attendees, topic, notes (rich text), action items
- [ ] Auto-create workpaper from interview notes
- [ ] Link to relevant engagement/finding

---

## Notes

- Fieldwork is the most time-intensive phase — UI must be fast and friction-free
- Offline support (Phase 7+) is critical here — auditors are often on-site without good internet
- The observation → finding escalation path is a key workflow (proven in AIMS v1)
