## Objective

Build the engagement planning module — planning memos, risk assessments, PRCM matrices, and work programs. Structure adapts based on the selected audit standard.

---

## Tasks

### 2.3.1 — Planning Document API
- [ ] CRUD for planning documents (one per engagement)
- [ ] Standard pack defines required planning sections
- [ ] Version tracking (draft iterations before approval)
- [ ] Approval workflow integration

### 2.3.2 — Planning Memo Form
- [ ] Dynamic form rendered from standard pack template
- [ ] Common sections (most standards share these):
  - Engagement objectives
  - Scope and subject matter
  - Methodology and approach
  - Risk assessment summary
  - Timeline and milestones
  - Resource requirements / budget hours
  - Key contacts
  - Prior audit results (link to previous engagement)
- [ ] Standard-specific sections:
  - GAGAS §7.05-7.10: 14-section structured memo
  - IIA: Planning considerations per Standard 2200
  - SOX: Scoping of significant accounts and controls
- [ ] Rich text editor for narrative sections
- [ ] Auto-save every 30 seconds
- [ ] Section-by-section completion indicator

### 2.3.3 — Risk Assessment
- [ ] Risk matrix (likelihood x impact = risk rating)
- [ ] Risk categories configurable per standard
- [ ] Inherent risk vs residual risk
- [ ] Risk-based audit selection (link to Audit Universe)
- [ ] Visual risk heat map

### 2.3.4 — PRCM Matrix (Process/Risk/Control Matrix)
- [ ] CRUD for PRCM entries:
  - Process/area
  - Risk description
  - Risk rating
  - Control description
  - Control type (Preventive/Detective/Corrective)
  - Control effectiveness
  - Test approach
- [ ] Link PRCM entries to audit tests and findings
- [ ] Import from previous engagement PRCM
- [ ] Export to PDF/Excel

### 2.3.5 — Work Program Management
- [ ] CRUD for work programs (multiple per engagement)
- [ ] Work program structure:
  - Objective
  - Scope
  - Procedure steps (ordered list with status, assignee, notes)
  - Linked workpapers
  - Linked findings
- [ ] Status tracking: Not Started → In Progress → Complete → Reviewed
- [ ] Assign procedures to specific team members
- [ ] Progress bar showing completion percentage
- [ ] Clone from template library

### 2.3.6 — Sampling Calculator
- [ ] Statistical sampling worksheet
- [ ] Sampling methods: Random, Systematic, Stratified, Judgmental
- [ ] Sample size calculator (confidence level, tolerable error, expected error)
- [ ] Sample selection and documentation
- [ ] Results evaluation

---

## Notes

- Planning is where the standard abstraction is most visible — GAGAS requires 14 specific sections, IIA requires different ones, SOX focuses on controls
- All planning data feeds into the final report (auto-populate objectives, scope, methodology)
- Risk assessment integrates with Audit Universe (Phase 5.1) for risk-based audit planning
