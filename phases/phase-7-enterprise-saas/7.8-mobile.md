# 7.8 — Mobile App (PWA / React Native)

**Parent**: [Phase 7 — Enterprise & SaaS](OVERVIEW.md)
**Estimated effort**: 4-5 days (PWA) or 6-8 weeks (React Native)

---

## Tasks

### Option A: Progressive Web App (Recommended for MVP)
- [ ] **PWA Configuration**: Service worker, manifest, offline-capable
- [ ] **Responsive Design**: All screens work on mobile (tablet priority)
- [ ] **Mobile-Optimized Views**:
  - Dashboard (simplified KPIs)
  - Engagement list with swipe actions
  - Finding detail (scrollable sections)
  - Approval queue (approve/reject with swipe)
  - Time entry (quick daily entry)
  - Camera integration (snap evidence, upload workpaper)
- [ ] **Offline Mode**: View cached data offline, queue changes for sync
- [ ] **Push Notifications**: Web push for approvals and deadlines
- [ ] **Install Prompt**: "Add to Home Screen" for app-like experience

### Option B: React Native App (Enterprise Tier)
- [ ] **Shared Logic**: Reuse validation schemas, API client, business rules
- [ ] **Native Features**: Camera, file system, biometric auth, push notifications
- [ ] **App Store Deployment**: iOS App Store + Google Play Store
- [ ] **MDM Support**: Mobile Device Management integration for enterprise clients

---

## Mobile Priority Features

1. **Approval queue** — Approve/reject on the go (most requested mobile feature)
2. **Time entry** — Quick daily time logging
3. **Evidence capture** — Take photos of documents, upload as workpapers
4. **Dashboard** — Quick status check
5. **Notifications** — Stay informed of deadlines and requests

---

## Notes

- Start with PWA — lower cost, same codebase, works immediately
- React Native for enterprise clients who require native app store presence
- Offline sync is critical for field auditors (government sites often have poor connectivity)
