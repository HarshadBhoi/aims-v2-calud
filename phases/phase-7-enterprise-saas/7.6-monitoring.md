# 7.6 — Monitoring, Logging & Alerting

**Parent**: [Phase 7 — Enterprise & SaaS](OVERVIEW.md)
**Estimated effort**: 2 days

---

## Tasks

- [ ] **Error Tracking**: Sentry for frontend and backend error tracking with source maps
- [ ] **Application Metrics**: Prometheus metrics (request duration, error rate, active users)
- [ ] **Dashboards**: Grafana dashboards for API performance, database queries, queue depth
- [ ] **Alerting**: PagerDuty/Slack alerts for: error spike, high latency, disk full, service down
- [ ] **Structured Logging**: JSON logs with request ID, tenant ID, user ID for traceability
- [ ] **Log Aggregation**: Centralized log storage (ELK, Loki, or CloudWatch)
- [ ] **Uptime Monitoring**: External uptime checks with status page
- [ ] **APM**: Application Performance Monitoring (New Relic or Datadog — optional)
- [ ] **Health Endpoints**: `/health` and `/ready` for load balancer checks
