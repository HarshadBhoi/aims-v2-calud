# 1.6 — File Storage Service

**Parent**: [Phase 1 — Foundation](OVERVIEW.md)
**Estimated effort**: 1-2 days

---

## Objective

Implement S3-compatible file storage for audit workpapers, evidence, and generated PDFs. Must support cloud (AWS S3, Azure Blob) and on-prem (MinIO).

---

## Tasks

### 1.6.1 — Storage Service
- [ ] Create `FileStorageService` using AWS S3 SDK (v3)
- [ ] S3-compatible configuration (works with AWS S3, Azure Blob via gateway, MinIO)
- [ ] Operations:
  - `upload(file, path)` — Upload file with metadata
  - `download(path)` — Get file stream
  - `getSignedUrl(path, expiresIn)` — Pre-signed URL for direct download
  - `delete(path)` — Remove file
  - `list(prefix)` — List files in folder
  - `copy(source, destination)` — Copy file (for versioning)

### 1.6.2 — File Organization
- [ ] Bucket/path structure:
  ```
  aims-{environment}/
  ├── {tenantId}/
  │   ├── workpapers/
  │   │   └── {engagementId}/
  │   │       └── {fileId}-{filename}
  │   ├── evidence/
  │   │   └── {capId}/
  │   │       └── {fileId}-{filename}
  │   ├── reports/
  │   │   └── {reportId}/
  │   │       └── {version}-{filename}.pdf
  │   └── exports/
  │       └── {exportId}.csv
  ```
- [ ] Tenant isolation at storage level (prefix-based)

### 1.6.3 — Upload Flow
- [ ] Frontend: drag & drop upload component with progress bar
- [ ] Backend: Multer middleware for file handling
- [ ] Virus scanning hook (ClamAV integration point — optional for Phase 1)
- [ ] File type validation (whitelist: pdf, docx, xlsx, pptx, jpg, png, csv, txt)
- [ ] File size limit (configurable per tenant, default 50MB)
- [ ] Store metadata in DB (Workpaper table), file in S3

### 1.6.4 — Download & Preview
- [ ] Pre-signed URL generation (expires in 15 minutes)
- [ ] Inline preview for PDFs and images
- [ ] Download with original filename
- [ ] Access control: only users with engagement access can download its workpapers

### 1.6.5 — Local Development
- [ ] MinIO in Docker Compose for local S3-compatible storage
- [ ] Auto-create bucket on startup
- [ ] MinIO console accessible at `localhost:9001`

---

## Notes

- All files encrypted at rest (S3 server-side encryption or MinIO encryption)
- File paths never contain user-controlled strings directly (use fileId, not filename)
- Workpaper versioning: keep all versions, mark current
- Consider CDN (CloudFront) for generated reports in production
