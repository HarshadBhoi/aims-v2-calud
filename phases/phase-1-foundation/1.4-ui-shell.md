## Design Principles

1. **Professional, not flashy** — Auditors need clean, information-dense layouts
2. **Information density** — Show more data, fewer clicks (auditors hate clicking through)
3. **Consistent patterns** — Every list page looks the same, every form behaves the same
4. **Accessible** — WCAG 2.1 AA compliance (Shadcn/Radix gives us this by default)
5. **Responsive** — Works on tablet (auditors in the field) but desktop-first

---

## Visual Reference (from AIMS v1)

Port the proven patterns from v1:
- Navy/indigo primary color scheme
- Card-based list views with status badges
- Stat chips at top of list pages
- Side panel for quick details
- Tab-based engagement detail view

Improve on v1:
- Better data tables (TanStack Table vs manual tables)
- Command palette for fast navigation
- Toast notifications instead of inline alerts
- Skeleton loading instead of spinners
- Dark mode option
