# 3.5 — GAGAS Report Generation

**Parent**: [Phase 3 — GAGAS Standard Pack](OVERVIEW.md)
**Estimated effort**: 3-4 days

---

## Objective

Port all 12 PDF report types from AIMS v1 to the new PDF engine, templated for GAGAS formatting requirements.

---

## Tasks

### 3.5.1 — PDF Report Engine Adaptation
- [ ] Create GAGAS report templates in `packages/standard-packs/gagas/reports/`
- [ ] Each template: pdfmake document definition (port from AIMS v1 PDFService.ts)
- [ ] Standard header/footer: organization name, report number, date, page numbers
- [ ] GAGAS compliance statement inclusion

### 3.5.2 — Individual Report Types
- [ ] **Audit Report** — Full engagement report (§6.02 structure)
- [ ] **Findings Register** — Summary of all findings with risk ratings
- [ ] **Annual Internal Audit Summary** — 15-section annual report (port from annualReportTemplate.ts)
- [ ] **Audit Planning Memo** — 14-section APM PDF
- [ ] **PRCM Matrix** — Process/Risk/Control matrix PDF
- [ ] **Testing Summary** — Audit test results summary
- [ ] **Sampling Worksheet** — Statistical sampling documentation
- [ ] **Work Program** — Procedures and completion status
- [ ] **Management Letter** — Findings summary for management
- [ ] **Follow-Up Report** — Status of prior recommendations
- [ ] **QA Review Summary** — QA checklist completion report
- [ ] **Independence Declarations** — Compiled independence forms

### 3.5.3 — Report Data Aggregation
- [ ] Service to aggregate data from multiple tables for reports:
  - Engagement details
  - Team members
  - Findings with four elements
  - Recommendations with management responses
  - CAP status
  - Time entries
  - QA results
- [ ] Caching for large reports
- [ ] Background generation for annual reports (BullMQ job)

### 3.5.4 — Report Preview & Download
- [ ] In-browser PDF preview
- [ ] Download as PDF
- [ ] Export as DOCX (future — using docx library)
- [ ] Report versioning (Draft v1, Draft v2, Final)
- [ ] Report storage in S3 for archival
