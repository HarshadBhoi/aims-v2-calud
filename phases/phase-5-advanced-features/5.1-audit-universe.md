# 5.1 — Audit Universe & Risk-Based Planning

> **⚠ Reconciliation 2026-05-01**: DEFERRED from Slice A (covered by Phase 5 OVERVIEW banner). The risk-assessment data model is now constrained by [ADR-0009](../../references/adr/0009-risk-assessment-history-table.md) — assessments are stored in a separate `RiskAssessment` table per (auditable entity, fiscal year), not as a JSONB blob on `AuditUniverseEntity`. The snapshot columns on the universe entity (`inherentRiskScore`, `residualRiskScore`, `lastRiskAssessment`) remain as a fast-path cache, kept in sync via trigger on assessment lock. Year-over-year trending is a query against the history table. See [`docs/06-design-decisions.md`](../../docs/06-design-decisions.md) §1.10.

**Parent**: [Phase 5 — Advanced Features](OVERVIEW.md)
**Estimated effort**: 3-4 days

## Tasks

- [ ] **Audit Universe Management**: CRUD for auditable entities (departments, programs, processes, systems)
- [ ] **Risk Assessment Matrix**: Inherent risk scoring (likelihood x impact) per entity
- [ ] **Risk Factors**: Configurable risk factors (financial materiality, complexity, prior findings, time since last audit, management concerns, regulatory changes)
- [ ] **Risk Scoring Engine**: Weighted calculation → prioritized audit universe
- [ ] **Annual Audit Plan**: Generate risk-based plan from universe (top-N by risk score)
- [ ] **Plan vs Actual Dashboard**: Track planned vs completed engagements
- [ ] **Multi-Year Planning**: Rolling 3-year audit plan with coverage tracking
- [ ] **Heat Map**: Visual risk heat map of audit universe
- [ ] **Carry Forward**: Unaudited entities carry forward with increased risk weight
